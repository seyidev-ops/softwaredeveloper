0.1 Who This Guide Is For

This guide assumes you have never written a single line of code. If you can type, save a file, and open a web browser, you have every prerequisite you need. By the end, you will be able to build complete, working web applications — front end and back end — and present yourself credibly as a junior software developer.

The promise is simple: every concept is explained in plain English, every concept is immediately followed by a hands-on practical you can copy, paste, and run, and every part ends with a real-world project that belongs in your portfolio.

0.2 The Golden Rules of Learning to Code

  1. Type it, then run it. Copy-paste gets you moving fast, but retype at least one practical per day by hand. Muscle memory is real.
  2. Break things on purpose. After every working practical, change something and predict what will happen. This is how professionals actually learn.
  3. Code every day. Ninety minutes daily beats ten hours on Saturday. Consistency rewires your brain; cramming does not.
  4. Errors are information, not failure. A red error message is the computer telling you exactly where to look. Read it slowly, from the top.
  5. Build, don’t just watch. Tutorials feel productive but build nothing. Every project in this guide produces something real you can show an employer.

0.4 Setting Up Your Developer Workstation

You need exactly three tools. All are free. Install them now, in this order.

Tool 1: Visual Studio Code (your code editor)

  1. Go to https://code.visualstudio.com and click the big download button for your operating system (Windows, Mac, or Linux).
  2. Run the installer. Accept all default options. On Windows, tick "Add to PATH" and "Open with Code" when offered.
  3. Open VS Code. Click the Extensions icon on the left sidebar (four squares). Search for and install: Live Server (by Ritwick Dey) and Prettier (code formatter).
WHY THIS MATTERS

Live Server lets you see your web page update instantly every time you save. Prettier keeps your code tidy automatically. Professionals use both daily.

Tool 2: Google Chrome (your testing browser)

Any modern browser works, but this guide uses Chrome because its built-in Developer Tools (DevTools) are the industry standard. Download it from google.com/chrome if you don’t have it. Memorise this shortcut now: press F12 (or Ctrl+Shift+I on Windows, Cmd+Option+I on Mac) to open DevTools. You will live in this panel.

Tool 3: Node.js (JavaScript outside the browser)

  1. Go to https://nodejs.org and download the version labelled LTS (Long Term Support).
  2. Run the installer with all default options.
  3. Verify it worked: open your terminal (Windows: press the Windows key, type "cmd", press Enter. Mac: open the Terminal app) and type the commands below, pressing Enter after each.
< p r e > < c o d e > n o d e - - v e r s i o n n p m - - v e r s i o n < / c o d e > < / p r e >

You should see version numbers like v22.x.x and 10.x.x. If you see "command not found", restart your computer and try again — the installer needs a restart to register itself.

Create Your Project Folder

Keep everything for this course in one folder. In your terminal, type:

< p r e > < c o d e > m k d i r d e v - j o u r n e y c d d e v - j o u r n e y m k d i r p a r t 1 - h t m l p a r t 2 - c s s p a r t 3 - j s p a r t 4 - d o m p a r t 5 - a s y n c p a r t 6 - n o d e p a r t 7 - c a p s t o n e p a r t 8 - i n t e g r a t i o n < / c o d e > < / p r e >

mkdir means "make directory" (create a folder) and cd means "change directory" (go into a folder). You just used your first real developer commands. Open the whole folder in VS Code: File → Open Folder → select dev-journey.

0.5 How Each Lesson Is Structured

  • Concept — a plain-English explanation with a real-world analogy.
  • Copy-and-Paste Practical — complete, runnable code. Never a fragment. Paste it, run it, watch it work.
  • Line-by-Line Breakdown — what each important line actually does.
  • Try This — a small modification challenge to make the knowledge stick.
  • Checkpoint Project — at the end of each part, a real-world project assembling everything learned.
