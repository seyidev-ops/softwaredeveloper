4.1 What Is the DOM?

When the browser loads your HTML, it builds a live, in-memory model of the page called the Document Object Model (DOM) — a family tree where every tag is a node JavaScript can find, read, change, create, or delete. This is the moment HTML, CSS, and JavaScript finally meet: HTML provides the elements, CSS the appearance, and JavaScript reaches into the DOM to change both in response to the user.

GOLDEN RULE

The <script> tag goes at the BOTTOM of <body> (or use the defer attribute). JavaScript cannot select an element that the browser has not drawn yet — scripts placed in <head> without defer find an empty page and fail silently.

4.3 Events: Responding to the User

An event is anything that happens: a click, a keypress, typing in a box, submitting a form, scrolling. You attach a listener — a function that runs when the event fires.

THE PROFESSIONAL PATTERN YOU JUST LEARNED

State → Render → Actions → Events. Keep your data in one place (the array), have one function that redraws the screen from that data, and have every user action change the data then call render. This exact pattern is the heart of React, Vue, and every modern framework. You just built it from scratch.

4.5 localStorage: Remembering Data After the Tab Closes

Right now your to-do list forgets everything on refresh. localStorage is a small permanent store inside the browser. It only stores text, so we convert with JSON.stringify (object → text) and JSON.parse (text → object).

4.7 Checkpoint Project: The Quiz App

Build this one with less hand-holding. Requirements:

  1. Store at least five questions as an array of objects: { question, options (array of 4), correctIndex }.
  2. Show one question at a time with its four options as buttons.
  3. Clicking an option shows green/red feedback, updates the score, and moves to the next question after one second (hint: setTimeout(fn, 1000)).
  4. At the end, show the score, a message based on performance (use if/else), and a Restart button.
  5. Stretch goal: save the best score in localStorage and display "Best: X".
