5.3 Asynchronous JavaScript: The Restaurant Analogy

Some operations take time: downloading data, reading files, querying databases. Synchronous code would freeze the whole page while waiting — imagine a waiter who stands frozen at the kitchen until your food is ready, ignoring every other table. Asynchronous code is the professional waiter: take the order, serve other tables, return when the kitchen calls.

A Promise is JavaScript’s "order ticket": an object representing a future result. It is pending while cooking, then either fulfilled (food arrives) or rejected (kitchen ran out of ingredients).

THE TWO RULES OF ASYNC/AWAIT

Rule 1: await may only appear inside a function marked async. Rule 2: always wrap awaits in try/catch — networks fail, servers go down, and professionals handle it gracefully instead of crashing.

5.4 fetch(): Getting Real Data From Real Servers

An API (Application Programming Interface) is a URL that returns data instead of a web page — almost always in JSON format, which looks exactly like JavaScript objects. fetch() is the built-in tool for calling APIs. This is the skill that turns your pages from toys into products.

5.6 Checkpoint Project: Live Weather App

Open-Meteo is a completely free weather API requiring no signup. Build an app that meets these requirements:

  1. Call the geocoding API to turn a city name into coordinates: https://geocoding-api.open-meteo.com/v1/search?name=Abuja
  2. Use the returned latitude/longitude in a second call: https://api.open-meteo.com/v1/forecast?latitude=9.06&longitude=7.49&current_weather=true
  3. Display city, temperature, and wind speed in a styled card.
  4. Show a friendly message while loading and a clear error if the city is not found.
  5. Stretch: save the last searched city in localStorage and load its weather automatically on page open.
