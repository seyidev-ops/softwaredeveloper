1.1 What Is HTML?

HTML stands for HyperText Markup Language. It is not a programming language — it is a markup language, which means it describes structure, not behaviour. Think of building a house: HTML is the skeleton and walls, CSS (Part 2) is the paint and decoration, and JavaScript (Part 3 onwards) is the electricity and plumbing that makes things actually work.

Every web page you have ever visited — Google, Facebook, your bank — is built on HTML. When your browser loads a page, it reads an HTML file and draws what it describes.

Line-by-Line Breakdown

  • <!DOCTYPE html> — tells the browser "this is a modern HTML5 document". Always the first line.
  • <html lang="en"> — the root container of everything. lang="en" tells screen readers and search engines the page is in English.
  • <head> — information ABOUT the page (title, character set). Nothing here is visible on the page itself.
  • <meta name="viewport" ...> — makes the page display properly on phones. Include it in every page you ever build.
  • <title> — the text shown in the browser tab.
  • <body> — everything visible lives here.
  • <h1> and <p> — a top-level heading and a paragraph.
THE ANATOMY OF A TAG

Most HTML elements have an opening tag <p>, content, and a closing tag </p> with a forward slash. The browser reads <p>Hello</p> as "this is a paragraph containing the word Hello". Forgetting a closing tag is the number one beginner bug.

1.4 Attributes: Giving Tags Extra Information

Attributes live inside the opening tag and follow the pattern name="value". You have already used several: href tells a link where to go, src tells an image where its file is, and alt provides text if the image fails to load (and for visually impaired users). Two attributes are so important they get their own introduction:

  • id="something" — a unique name for ONE element on a page. Like a national ID number — no two elements may share it. JavaScript will use ids constantly.
  • class="something" — a reusable label that MANY elements can share. Like a uniform — everyone in class "card" can be styled the same way by CSS.

1.5 Semantic HTML: Writing Pages That Mean Something

You could build an entire site out of <div> boxes, but professionals use semantic tags — tags whose names describe their purpose. Search engines rank semantic pages higher, screen readers navigate them better, and other developers understand them faster.

Semantic TagPurpose
<header>Top section of a page or article — logo, title, navigation
<nav>Navigation links
<main>The unique main content of the page (one per page)
<section>A themed grouping of content, usually with a heading
<article>Self-contained content: a blog post, a job listing, a product card
<aside>Side content: related links, adverts, author bio
<footer>Bottom of page: copyright, contact, social links

Key Points

  • <label for="x"> connects to the input whose id is x — clicking the label focuses the input. Always pair them.
  • type="email" and required give you free validation: the browser refuses to submit an empty or malformed email.
  • Radio buttons sharing the same name form a group where only one can be selected.
  • Common input types: text, email, password, number, tel, date, checkbox, radio, file, color, range. Try swapping types and watch the browser change behaviour.

1.8 Checkpoint Project: Your Personal Profile Page

This is your first real-world deliverable — the seed of the portfolio you will deploy in Part 9. Build it entirely yourself, using the template below as a starting skeleton.

Requirements

  1. A header with your name and a navigation bar linking to each section (hint: <a href="#about"> jumps to the element with id="about").
  2. An About section with your photo, a short bio, and a list of your goals.
  3. A Skills section using a table: skill name, level, and learning status.
  4. A Projects section with at least two <article> cards (use placeholder text for now — you will fill these with real projects soon).
  5. A Contact section with a working form: name, email, message, submit button.
  6. A footer with the current year and links to your social profiles.
